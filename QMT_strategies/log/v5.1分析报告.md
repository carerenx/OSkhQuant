## 二、🔴 根因：QMT 回测使用不复权价格

### 价格对比

| 代码 | OSkhQuant | QMT V5.1 | 倍率 |
|------|-----------|----------|------|
| 000988.SZ | 300.39 | 167.89 | 1.79x |
| 600895.SH | 400.33 | 33.41 | 11.98x |
| 600418.SH | 76.35 | 28.97 | 2.63x |
| 600348.SH | 163.15 | 8.17 | 19.97x |

> OSkhQuant 使用 `khHistory(..., fq="pre")` — **前复权**  
> QMT `get_history_data` — **不复权**（默认）

### 不复权的后果

```
买入日:  000807.SZ @ 24.64 (不复权)
次日:    同只股票除权除息 → "市价" 11.98 (不复权)
止损判断: (11.98 / 24.64 - 1) = -51.4% > 25% 止损线 → 触发止损

真实情况: 前复权下价格可能只跌了 2%，完全不该止损
```

### 数据验证

| 指标 | 数值 | 说明 |
|------|------|------|
| **1天内被止损** | **1,957笔 (86.5%)** | 除权除息日的价格"暴跌"触发止损 |
| **2天内被止损** | 32笔 | |
| sell/buy 价格比 | 中位数 **0.56x** | 买入后"暴跌"44% |
| sell/buy 极值 | 0.09x ~ 2.59x | 物理上不可能的单日涨跌 |

## 三、🟡 次要差异

### 3.1 股票池不同

| | OSkhQuant V5 | QMT V5.1 |
|---|---|---|
| 来源 | 配置文件 500只 | `get_stock_list_in_sector` 487只 |
| 首日选股 | 601099, 603087, 000988... | 000807, 000975, 600027... |
| 重叠 | — | **仅 4/12 重叠** |

原因：QMT 实时获取 CSI500 成分股，2026年的成分股列表与配置文件中固定的 500 只不同（成分股每年调整）。

### 3.2 MA60 基准不同

- OSkhQuant: bar=1 开始，前 60 天 MA60 不准确（数据不足时返回 'ok'）
- QMT: bar=5337 开始，MA60 使用 2023年10-12月真实数据
- 导致 QMT 更早触发软防御（86天 vs OSkhQuant 更少）

## 四、修复方案

### 方案 A：QMT 全局设置（推荐，最简单）

在 QMT 客户端：**设置 → 数据管理 → K线复权方式 → 前复权**

然后重新下载历史数据（清空缓存 → 重新下载）。

### 方案 B：策略中使用 xtdata 替代 get_history_data

如果方案 A 不可行，可以修改策略使用 xtdata：

```python
# 在 handlebar 开头替代 get_history_data：
from xtquant import xtdata

current_date = timetag_to_datetime(ContextInfo.get_bar_timetag(bar), '%Y%m%d')
hist_close = xtdata.get_market_data_ex(
    field_list=['close'], stock_list=all_codes,
    period='1d', start_time='20100101', end_time=current_date,
    dividend_type='front'  # 前复权
)
```

但这会显著增加代码复杂度和运行时间。

### 方案 C：使用 ContextInfo 的前复权参数

部分 QMT 版本支持：
```python
ContextInfo.get_history_data(need_bars, '1d', 'close', dividend_type='front')
```
需要验证当前 QMT 版本是否支持此参数。

## 五、结论

| 问题 | 严重度 | 状态 |
|------|--------|------|
| V5.0 P&L 恒为 0% | 🔴 致命 | ✅ 已修复 |
| V5.1 不复权价格 | 🔴 致命 | ❌ 待修复 |
| 股票池不同 | 🟡 中等 | 次要，影响有限 |
| MA60 基准 | 🟡 轻微 | 选股结果不同 |

**V5.1 的 P&L 计算已修复（不再全是 0%），但价格数据是**不复权**的。这是 QMT `get_history_data` 的默认行为。除非 QMT 中开启了前复权，否则价格因为除权除息出现物理上不可能的单日跳变（-82%、+159%），导致 86.5% 的持仓在 1 天内被误触发止损。**

**核心行动：在 QMT 客户端设置中开启前复权，清除本地数据缓存后重新下载历史数据，再重新运行回测。**
